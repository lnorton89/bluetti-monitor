Goal: Make the existing dashboard genuinely usable from a phone browser on the local network.

This plan adapts the existing desktop dashboard layout for phone-sized screens (320px-428px viewport width).
</context>

<objective>
Adapt the existing dashboard CSS and components to work well on phone browsers. Ensure core monitoring data is readable, navigation is accessible, and information density is appropriate for small screens.
</objective>

- dashboard/src/index.css (existing mobile breakpoints at 1024px and 480px)

<action>
## Task 1: Audit and Fix 480px Mobile Breakpoint

Read dashboard/src/index.css and identify any issues with the existing 480px breakpoint styles.

Expected state after fix:
- All grid layouts collapse to single column (summary-grid, detail-grid, category-grid)
- Route hero section stacks with stats below copy
- Power flow panel shows vertical stack (Input → Battery → Output)
- All touch targets (buttons, chips, nav links) have adequate spacing
- Typography scales appropriately with clamp() functions
- No horizontal overflow issues

## Task 2: Verify Sidebar Drawer Mobile Behavior

Test the existing sidebar drawer at 480px:
1. Verify hamburger button is visible and tappable
2. Verify sidebar opens as drawer from left with overlay
3. Verify close button works
4. Verify navigation links work and close drawer on tap

Expected: All navigation works from phone browser.

## Task 3: Test Core Monitoring Views on Mobile

Verify each page renders correctly on phone:
1. Overview page - hero section, power flow, device status readable
2. Charts page - chart containers resize properly
3. Solar page - solar inputs stack vertically
4. Raw Data page - table scrolls horizontally with touch

Expected: All four views usable on 320px-428px viewport.
</action>

1. `dashboard/src/index.css` contains valid 480px mobile breakpoint

5. Power flow panel shows Input → Battery → Output vertically

10. All four navigation routes (Overview, Charts, Solar, Raw Data) accessible from phone
