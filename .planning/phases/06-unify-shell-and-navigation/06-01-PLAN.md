<action>
Create `dashboard/src/store/shell.ts` with a tiny Zustand store that owns the current mobile route signal value for the active route:

- `routeId: 'overview' | 'charts' | 'solar' | 'raw' | null`
- `value: string`
- `setRouteSignal(routeId: string, value: string): void`
- `resetRouteSignal(routeId?: string): void`

Add stable selectors to the existing shell surfaces without waiting for the full thin-shell refactor:

- `data-testid="shell-title"` on the top-bar route identity element
- `data-testid="shell-route-signal"` on the top-bar single-signal surface that Phase 6 will populate
- `data-testid="sidebar-nav"` on the nav container
- `data-testid="sidebar-overlay"` on the overlay
- `data-testid="sidebar-route-overview"`, `sidebar-route-charts`, `sidebar-route-solar`, and `sidebar-route-raw` on the nav links

Keep the selectors stable and tied to the registry-backed route contract so the later Playwright rewrite can target them directly. `App.tsx` should derive the visible signal label from the route registry, not from the store payload.
</action>
