## Shell Contract

- Use a thin persistent shell only. The shell owns navigation, connection state, freshness, battery, device count, and notification readiness.
- Remove the persistent route hero from the shell. Route pages own their own page header and contextual framing.
- Keep one route registry as the source of truth for path, label, icon, mobile title, and mobile single-signal mapping.
- Keep desktop navigation as a fixed left sidebar.
- Keep mobile navigation as a drawer version of the same sidebar content. Do not add a second mobile nav model.
- Show exactly one shell-level page identity at a time. Desktop may show route title in the top bar; mobile shows page title plus one route-relevant signal.
- Use these mobile route signals:
- `Overview`: battery reserve percent
- `Charts`: active history window
- `Solar`: live solar input watts
- `Raw Data`: visible field count

---

## Navigation Contract

- Desktop sidebar width: 200px.
- Mobile drawer width: `min(280px, calc(100vw - 56px))`.
- Top bar stays sticky.
- Hamburger, drawer close, and nav rows must stay at or above 44px touch height on mobile.
- Active route state must be visible in both desktop and mobile nav through the same label, icon, and highlighted container treatment.
- Drawer closes on overlay click, close button click, and route selection.
- Page identity must not be duplicated between shell and page body. The shell provides route name; the page provides route-specific explanation and content framing.

| md | 16px | Default element spacing |

| xl | 32px | Layout gaps |

## Color

| Dominant (60%) | #0a0c0e | App background and main canvas |
| Secondary (30%) | #111418 | Sidebar, top bar, cards, drawer, nav surfaces |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Open Navigation |
| Empty state heading | Waiting for live telemetry |
| Empty state body | Start `bluetti-mqtt`, confirm the broker connection, and the dashboard will populate automatically. |
| Error state | Connection lost. Check the bridge or API path, then wait for the live stream to reconnect. |
| Destructive confirmation | none in this phase |

---

## Responsive Behavior

- Desktop breakpoint: sidebar fixed and always visible above 1024px.
- Tablet and below: sidebar becomes drawer; content column spans full width.
- Mobile top bar may wrap status content, but only one route-relevant signal is allowed alongside page identity.
- Do not stack a full status cluster above route content on mobile.
- Route pages must begin with their own header block once the shell hero is removed.

---

## Motion

- Drawer slide duration: 200-300ms.
- Overlay fade duration: 200ms.
- Hover lift on shell controls: max 1px translate.
- Do not animate page headers as hero panels in this phase.

- Preserve current fonts and color variables for this phase so shell cleanup does not expand into a full visual-system rewrite.
