  <action>
Refactor Overview.tsx to replace local inline component implementations with the shared surface components from Plan 01.

**Specific replacements:**

1. **`SummaryCard`** → Replace with `<MetricTile>` + keep the `.summary-grid` layout class (or switch to `<div className="tile-grid tile-grid--fit">`).
   - `SummaryCard` currently takes `label`, `value`, `detail?`, `accent?` — all of which map directly to MetricTile props
   - The `.summary-card` CSS class can be removed after confirming MetricTile's `.metric-tile` rendering matches visually. Keep `.summary-card-value` large font rendering by adding a `className` override on MetricTile or accepting a shared "large metric" variant.

2. **`InfoTable`** → Replace with `<SectionPanel title={title} icon={Icon}>` containing a `.info-table` div with `<InfoRow>` items.
   - `InfoTable` currently renders `<Card className="info-panel">` + header + `.info-table` div with `.info-row` items
   - SectionPanel gives the Card + title pattern; InfoRow gives the key-value rows

3. **`StatPanel`** → Replace with `<SectionPanel title={title} icon={Icon}>` containing a `<div className="tile-grid tile-grid--cols-2">` of `<MetricTile>` items.
   - StatPanel renders card + header + `.stat-grid` of `.stat-tile` items
   - Each tile has label + value + optional accent → maps to MetricTile

4. **Offline banner** → Replace inline `.offline-banner` div with `<StatusChip variant="warning">` or a dedicated `<OfflineBanner>` component that composes StatusChip. Actually, keep the banner more prominent since it's a full-width urgent signal. Instead, wrap it in a shared `offline-banner` class but use StatusChip for the retry button.

5. **Stale indicator** → Replace inline `.stale-indicator` with `<StatusChip variant={staleSeverity === 'stale' ? 'error' : 'warning'}>` or keep the indicator as-is since it already works well. The goal per UI-09 is *reusing patterns*, not necessarily replacing every inline element with an identical component if the inline version already matches the token classes.

6. **Device status strip** → The `.device-status-pill` can become `<StatusChip>` with appropriate variant mapping.

7. **Switchboard section** → The bool badges in the switchboard grid can stay as `BoolBadge` since they're already a shared component. The card wrapper becomes `<SectionPanel>` or `<Card className="surface-card">`.

**Keep these local to Overview** (they are too domain-specific to generalize):
- `Hero` component with its power-flow panel, battery bar, and mode chip — this is highly specific to the overview page
- `Ac500Overview` layout logic and device data extraction helpers
- `describeActivity`, `batteryTone`, `modelName`, `deviceSerial` — domain helpers

**Import changes:**
Remove local `SummaryCard`, `InfoTable`, `StatPanel` component definitions.
Add: `SectionPanel, MetricTile, InfoRow, StatusChip, PageHeader` to imports from `'../components/ui'`.

**CSS cleanup in index.css:**
After the Overview refactor, `.summary-card`, `.summary-card-label`, `.summary-card-value`, `.summary-card-detail`, `.info-panel`, `.stat-grid`, `.stat-tile`, `.stat-tile-label`, `.stat-tile-value`, `.device-status-pill`, `.device-status-dot` can be flagged as deprecated aliases that still render (via comma selectors with token classes). Remove them only if a grep confirms no remaining usage.
