**What to extract:**

1. **RANGE_PRESETS** — Both Charts.tsx and Solar.tsx define essentially the same range preset array:
   ```ts
   const RANGE_PRESETS = [
     { id: '1h', label: '1H', minutes: 60, limit: 720, bucketMs: 60_000 },
     { id: '6h', label: '6H', minutes: 360, limit: 2_400, bucketMs: 5 * 60_000 },
     { id: '24h', label: '24H', minutes: 1_440, limit: 5_000, bucketMs: 15 * 60_000 },
     { id: '72h', label: '3D', minutes: 4_320, limit: 5_000, bucketMs: 60 * 60_000 },
   ] as const;
   ```
   Extract this to `shared-controls.ts` and export it as `RANGE_PRESETS` with a `RangePreset` type derived from it.

2. **useDeviceSelector hook** — Both Charts and Solar have the same device selection pattern:
   ```ts
   const [selectedDevice, setSelectedDevice] = useState(liveDevices[0] ?? '');
   useEffect(() => {
     if (liveDevices.length === 0) { setSelectedDevice(''); return; }
     if (!selectedDevice || !liveDevices.includes(selectedDevice)) {
       setSelectedDevice(liveDevices[0]);
     }
   }, [liveDevices, selectedDevice]);
   ```
   Extract this into a `useDeviceSelector(devices: string[])` hook that returns `[selectedDevice, setSelectedDevice]`.

3. **Type exports** — Export `RangePreset` type so consumers don't need to repeat the `as const` inference.
